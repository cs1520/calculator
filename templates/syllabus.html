Programming Languages for Web Applications

Course Description

This course will focus on Web programming tools and technologies, both from the server side and the client side point of view. Server side technologies include Python and HTTP protocols. Client side technologies include the JavaScript scripting language (including JavaScript libraries), HTML, CSS, and AJAX protocols.

Prerequisites: CS445

Graded Material

Class Project

There will be a class project throughout the term. This project will be completed in groups. Some basic requirements for the project:

  • Your project must include at least one stylesheet with appropriate usage.
  • It must utilize the Google App Engine Users service or another user system of your choice.
  • Your project must have plenty of meaningful Python code (minimum of 150 lines per group member).
  • Your web application must use some dynamic JavaScript (minimum of 80 lines per group member) to interact with a client.
  • It must utilize a Google App Engine data storage service in a meaningful way.
  • Your project will need to be viewable by all class members.
  • All group members must collaborate and all group members must write code.
  • Your project must use at least one additional API (AppEngine services or other APIs).
  • Your group must report reasonable progress on the project at every progress report session.

Project groups will be 4 students. We will have time in class to come up with groups, but I will assign groups if necessary. Neither groups nor topics will not need to be decided in the first week.

The project is worth 35% of the grade and is distributed as follows:

Project Proposal & Plan 2%
Completion of Peer Reviews (3 @ 1% each) 3%
Completion of Project Reviews (3 @ 1% each) 3%
Peer Feedback (3 @ 4% each) 12%
Milestone Progress (3 @ 4% each) 12%
Final Project Evaluation 8%
Total 40%

Milestone Presentations will be given in class at 3 points throughout the semester. The point of these presentations is to report on the work you’ve done so far, share your project with the class, get some feedback, and make sure you stay on track. Note that only one group member will need to present the project progress.

For each project milestone presentation, you will need to submit Peer Feedback for everyone in your group. You will receive 1 point for each Completed Peer Review (i.e., the ones you submit), and the Peer Feedback grade will be based on your peer feedback. You'll also be expected to complete Project Reviews of the project presentations at each review.

The Final Project Evaluation will be based on review of your source code, neatness, completeness, and how well your project meets the stated requirements (as well as any other requirements throughout the term).

In order to avoid bad group situations, in addition to peer reviews, ONE TIME in the term up until the second review, you can do one of the following:

  • Remove one person from your group. This must be done by unanimous decision by the rest of the group members. Group members that are removed are free to join another group, band together, or complete the project independently with a copy of the current code.
  • Trade one person from your group for another person in another group. This must be done by unanimous decision of all members in both groups.
